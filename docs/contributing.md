# Contributing to Astra

Thank you for your interest in contributing to Astra! We aim to build the best full-stack Go framework with a focus on developer experience and production readiness.

## Code of Conduct

Please be respectful and professional in all interactions. We follow the standard Contributor Covenant.

## How to Contribute

1.  **Report Bugs**: Use GitHub issues to report bugs. Provide a clear description and a reproduction if possible.
2.  **Suggest Features**: Open an issue to discuss new features before implementing them.
3.  **Submit Pull Requests**:
    *   Fork the repository.
    *   Create a feature branch (`git checkout -b feature/my-new-feature`).
    *   Commit your changes (`git commit -am 'Add some feature'`).
    *   Push to the branch (`git push origin feature/my-new-feature`).
    *   Create a new Pull Request.

## Development Setup

1.  Requires Go 1.22+.
2.  Install dependencies: `go mod download`.
3.  Run tests: `go test ./...`.
4.  Run the local dev server (if applicable): `astra dev`.

## Coding Standards

*   Follow standard Go formatting (`gofmt`).
*   Write unit tests for new features.
*   Document public APIs.
*   Keep the "Rails/AdonisJS" ergonomics in mind.
